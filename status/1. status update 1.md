## 4. BITÁCORA DE ERRORES Y SOLUCIONES (TROUBLESHOOTING)
El cruce entre Windows, la máquina virtual y el host Podman generó múltiples obstáculos ya superados. Revisar esta tabla si el proyecto falla en un nuevo entorno local:

| Error / Comportamiento | Causa Diagnosticada | Solución Implementada |
|---|---|---|
| **Fallo compilación `mvnw` (Windows)** | Faltaba la carpeta oculta `.mvn` y el `maven-wrapper.properties` al mover/clonar el proyecto. | Recreación manual vía comandos en PowerShell. |
| **Fallo NetBeans `${start-class}`** | El IDE usa `exec-maven-plugin` asumiendo rutas erróneas para el método `main`. | Se abandonó el IDE para arranque. Ejecutar estrictamente por consola: `.\mvnw spring-boot:run`. |
| **`Port 8080 was already in use`** | Colisión del puerto interno con los servicios web legacy. | Cambio preventivo de puerto en properties: `server.port=8081`. |
| **`Connection refused` (127.0.0.1)** | La VM de Windows buscaba la base de datos localmente dentro de sí misma, ignorando al host Podman. | Apuntar la ruta JDBC explícitamente a la IP del contenedor: `10.10.1.1`. |
| **`Schema-validation: missing table`** | Spring asume nombres de tabla en minúscula y busca en un esquema por defecto inexistente. | Inyección de reglas `physical-strategy` y `default_schema=dbo` en properties. |
| **Error parámetro 'X' en terminal** | PowerShell nativo intercepta la palabra `curl` como un alias de su comando `Invoke-WebRequest`. | Instalar ejecutable `curl` real y llamarlo estrictamente como `curl.exe`. |
| **`404 Not Found` en Postman/curl** | El servidor arrancó pero el archivo del `Controller` modificado no se había guardado en el IDE. | Guardar archivos activamente y reiniciar el proceso Spring Boot. |

## 5. GUÍA DE DESPLIEGUE LOCAL (NUEVA MÁQUINA)
1. Instalar SQL Server. Ejecutar el script base para crear `MimosDemo`.
2. Crear usuario de acceso local: 
   `CREATE LOGIN api_mimos_user WITH PASSWORD = 'MimosDev2026*';`
   `USE [MimosDemo]; CREATE USER api_mimos_user FOR LOGIN api_mimos_user;`
   `ALTER ROLE db_owner ADD MEMBER api_mimos_user;`
3. Ajustar `spring.datasource.url` en el archivo `application.properties`. Si la base de datos es nativa en Windows (sin Podman/VM), cambiar `10.10.1.1` por `127.0.0.1`.
4. En la terminal de consola, dentro de la ruta `/api_rest/`, ejecutar: `.\mvnw spring-boot:run`.
