## 2. HITOS ALCANZADOS EN ESTA SESIÓN

### Entidad `Categoria` — completada y validada localmente
Esta es la tercera entidad del sistema. Al igual que `Rol`, es una tabla de catálogo independiente sin llaves foráneas, fundamental para la posterior creación de la entidad `Producto`.

Se generaron e integraron los 5 archivos de la arquitectura:

| Archivo | Ruta | Responsabilidad |
|---|---|---|
| `Categoria.java` | `entity/` | Mapea la tabla `Categorias` (id, nombre, descripcion). |
| `CategoriaRepository.java` | `repository/` | Hereda de `JpaRepository` para operaciones CRUD sin escribir SQL. |
| `CategoriaDTO.java` | `dto/` | Objeto de transferencia de datos seguro para respuestas HTTP. |
| `CategoriaService.java` | `service/` | Lógica de negocio y mapeo de Entity a DTO (`toDTO`). |
| `CategoriaController.java` | `controller/` | Endpoints HTTP REST bajo la ruta `/api/categorias`. |

### Decisiones técnicas tomadas y por qué

**Un solo DTO (`CategoriaDTO`) en lugar de múltiples**
A diferencia de `Usuario` (que requirió `CrearUsuarioDTO` para proteger el `password_hash`), `Categoria` no maneja datos sensibles ni campos complejos auto-gestionados por la base de datos (más allá del ID). Por lo tanto, un único DTO es suficiente tanto para enviar respuestas como para estructurar el modelo mental, manteniendo el código limpio.

**Transición oficial de Curl (PowerShell) a Postman**
Durante las pruebas del método POST, PowerShell demostró ser inherentemente problemático para parsear JSONs escapados por línea de comandos (ej. arrojando `Unexpected character '' (code 92)`). Se decidió establecer **Postman** como la herramienta oficial y obligatoria para probar endpoints que requieran `@RequestBody` (POST/PUT), evitando perder tiempo de desarrollo peleando contra la consola de Windows.

### Endpoints disponibles y validados

| Método | URL | Descripción | Código respuesta |
|---|---|---|---|
| `GET` | `/api/categorias` | Lista todas las categorías (Helados, Postres, Bebidas) | 200 |
| `GET` | `/api/categorias/{id}` | Busca categoría por ID | 200 / 404 |
| `POST` | `/api/categorias` | Registra una nueva categoría | 201 |
| `DELETE` | `/api/categorias/{id}` | Elimina una categoría | 204 |

## 3. BITÁCORA DE ERRORES Y SOLUCIONES (ACUMULADA)

| Error / Comportamiento | Causa Diagnosticada | Solución Implementada |
|---|---|---|
| **Fallo NetBeans `${start-class}`** | El IDE asume rutas erróneas. | Ejecutar siempre por consola: `.\mvnw spring-boot:run`. |
| **`Connection refused` (127.0.0.1)** | La VM buscaba la DB localmente. | JDBC apunta a `10.10.1.1` (Podman). |
| **`Schema-validation: missing table`** | Hibernate asume nombres en minúscula. | `physical-strategy` estándar en properties. |
| **`JSON parse error: Unexpected character ''`** | PowerShell interpola mal las comillas y barras invertidas en el `-d` de curl. | **(NUEVO)** Abandonar `curl` para peticiones con JSON complejo. Usar **Postman** (Body > raw > JSON). |
