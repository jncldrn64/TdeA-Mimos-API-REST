## 2. HITOS ALCANZADOS EN ESTA SESIÓN

### Entidad `Usuario` — completada y validada en producción
Esta es la entidad central del sistema: todos los roles, carritos, ventas y PQRS dependen de ella.

Se generaron e integraron los 5 archivos de la arquitectura:

| Archivo | Ruta | Responsabilidad |
|---|---|---|
| `Usuario.java` | `entity/` | Mapea la tabla `Usuarios` con FK a `Roles` |
| `UsuarioRepository.java` | `repository/` | CRUD + `findByEmail()` personalizado |
| `UsuarioDTO.java` | `dto/` | Respuesta pública (sin `password_hash`) |
| `CrearUsuarioDTO.java` | `dto/` | Entrada del POST (campos que envía el cliente) |
| `UsuarioService.java` | `service/` | Lógica de negocio + encriptación BCrypt |
| `UsuarioController.java` | `controller/` | Endpoints HTTP REST |

### Decisiones técnicas tomadas y por qué

**`@ManyToOne(fetch = FetchType.LAZY)` en la FK a Rol**
Un usuario pertenece a un rol. `LAZY` significa que Hibernate no carga el objeto `Rol` a menos que se necesite explícitamente, evitando consultas innecesarias a la DB en cada lectura de usuario.

**Dos DTOs en lugar de uno (`UsuarioDTO` + `CrearUsuarioDTO`)**
El DTO de salida (`UsuarioDTO`) nunca incluye `password_hash`. El DTO de entrada (`CrearUsuarioDTO`) acepta la contraseña en texto plano para que el Service la encripte antes de guardar. Mezclarlos en un solo objeto abriría el riesgo de exponer el hash en la respuesta JSON.

**Encriptación con BCrypt en el Service**
`BCryptPasswordEncoder` transforma la contraseña antes del INSERT. El hash resultante es irreversible: si la DB es comprometida, las contraseñas no se exponen. Se agregó la dependencia `spring-security-crypto` al `pom.xml` (módulo liviano, no activa Spring Security completo).

**`insertable = false, updatable = false` en `estado` y `createdAt`**
Estos campos tienen valores `DEFAULT` en SQL Server (`'activo'` y `GETDATE()`). Marcándolos así, Hibernate no los incluye en el INSERT y deja que la DB los asigne sola. El `null` que aparece en la respuesta JSON es esperado — Hibernate no re-lee la fila después de insertar. No es un bug.

**`findByEmail()` en el Repository**
Spring genera el SQL automáticamente a partir del nombre del método. Se usa en el Service para verificar unicidad del email antes de insertar, devolviendo un error 400 claro en lugar de dejar que la DB explote con un error de constraint UNIQUE.

### Endpoints disponibles y validados

## 3. BITÁCORA DE ERRORES Y SOLUCIONES (ACUMULADA)

Incluye los errores del estado anterior más los nuevos de esta sesión:

| Error / Comportamiento | Causa Diagnosticada | Solución Implementada |
|---|---|---|
| **Fallo compilación `mvnw` (Windows)** | Faltaba la carpeta oculta `.mvn` y el `maven-wrapper.properties`. | Recreación manual vía comandos en PowerShell. |
| **Fallo NetBeans `${start-class}`** | El IDE usa `exec-maven-plugin` con rutas erróneas. | Ejecutar siempre por consola: `.\mvnw spring-boot:run`. |
| **`Port 8080 was already in use`** | Colisión con servicios web legacy. | `server.port=8081` en `application.properties`. |
| **`Connection refused` (127.0.0.1)** | La VM buscaba la DB localmente, ignorando al host Podman. | JDBC apunta a `10.10.1.1`. |
| **`Schema-validation: missing table`** | Hibernate asume nombres en minúscula y esquema por defecto inexistente. | `physical-strategy` estándar + `default_schema=dbo` en properties. |
| **`curl` no funciona en PowerShell** | PowerShell intercepta `curl` como alias de `Invoke-WebRequest`. | Usar `curl.exe` explícitamente. |
| **`404 Not Found` en Postman/curl** | Archivo del Controller no guardado antes de reiniciar. | Guardar archivos y reiniciar Spring Boot. |
| **`package ...bcrypt does not exist`** | Dependencia `spring-security-crypto` no estaba en `pom.xml`. | Agregar la dependencia y correr `.\mvnw dependency:resolve`. |
| **`JSON parse error: Unexpected character '\'`** | PowerShell interpola las comillas escapadas del JSON en `-d`. | Usar variable: `$body = '{...}'` y pasar `-d $body`. |

## 6. GUÍA DE DESPLIEGUE LOCAL (NUEVA MÁQUINA)

1. Instalar SQL Server. Ejecutar el script base para crear `MimosDemo`.
2. Crear usuario de acceso:
   ```sql
   CREATE LOGIN api_mimos_user WITH PASSWORD = 'MimosDev2026*';
   USE [MimosDemo];
   CREATE USER api_mimos_user FOR LOGIN api_mimos_user;
   ALTER ROLE db_owner ADD MEMBER api_mimos_user;
   ```
3. Ajustar `spring.datasource.url` en `application.properties`. Sin Podman/VM, cambiar `10.10.1.1` por `127.0.0.1`.
4. Desde `/api_rest/` en consola: `.\mvnw spring-boot:run`
